  - Outline: ✓ 35个章节数据正常
  - Summaries: ✗ **0个摘要（实际应该是35个）**
  - Highlight: ✓ 15个亮点数据正常

### 4. 根本原因分析

#### 问题1: Pydantic模型验证失败
**文件**: `app/models.py:94`

```python
# 修复前
cited_segment_ids: List[int] = Field(..., min_items=0, max_items=20, description="引用的段落 id")
```

**问题**:
- 某些章节引用了超过20个段落（实际数据: ch7引用了131个段落）
- Pydantic验证失败: `List should have at most 20 items after validation, not 131`
- 导致整个 `ChapterSummary` 对象创建失败，summaries列表为空

**修复**:
```python
# 修复后
cited_segment_ids: List[int] = Field(..., min_items=0, max_items=200, description="引用的段落 id")
```

#### 问题2: HighlightItem同样存在约束问题
**文件**: `app/models.py:102`

**问题**:
- 实际数据中某些亮点引用了128个段落
- 模型注释说"约定1-5个"，但实际数据远超这个范围

**修复**: 移除了过于严格的注释描述

#### 问题3: Highlight加载逻辑错误
**文件**: `app/main.py:1923-1963`

**问题**:
- `DerivedDataRepository.get()` 返回的 `highlight_data["highlights"]` 是完整的 HighlightCard 字典
- 包含: tldr_zh, worth_listening_verdict, highlights数组等
- 但代码错误地假设 `highlight_data["highlights"]` 是一个数组

**修复**:
```python
# 修复前
if "highlights" in highlight_data:
    highlight_data["highlights"] = [
        HighlightItem(...) for h in highlight_data["highlights"]  # 错误：试图遍历字典
    ]

# 修复后
if highlight_data and "highlights" in highlight_data:
    card_data = highlight_data["highlights"]  # 获取完整的 HighlightCard 字典
    if "highlights" in card_data and isinstance(card_data["highlights"], list):
        card_data["highlights"] = [
            HighlightItem(...) for h in card_data["highlights"]  # 正确：遍历实际的数组
        ]
    highlight = HighlightCard(**card_data)
```

## 技术要点

### Pydantic验证失败的影响
- 当列表项数量超过约束时，整个模型创建失败
- 异常被静默捕获（注释掉的logger），导致返回空数据
- 一个字段的验证失败会影响整个数据加载

### 数据结构设计的注意事项
- Repository层的数据结构与业务模型可能存在嵌套差异
- `DerivedDataRepository.get()` 将JSON字段解析后添加到字典中
- 需要明确区分"JSON字段名"（highlights_json）和"解析后的字段名"（highlights）

### 数据库存储格式
```
表: highlight
├── episode_id: TEXT
├── highlights_json: TEXT  ← 存储完整的 HighlightCard JSON（包含tldr_zh, highlights数组等）
└── updated_at: TEXT
```

- 重新评估模型约束的必要性
- 考虑使用软约束（警告而非错误）
- 为不同场景设计不同的验证规则

### 4. 测试覆盖
- 添加边界情况的测试
- 测试Pydantic验证失败的场景
- 添加API集成测试
