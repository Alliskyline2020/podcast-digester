## 技术细节

### 标点添加策略
1. **合并**: 将相邻段落（<3秒间隔）合并成句子
2. **批处理**: 每批100个段落，平衡速度和成本
3. **LLM调用**: 使用DeepSeek API添加标点
4. **分配**: 将标点文本按字符比例分配回原始段落

### 数据流程
```
原始字幕
  → 合并成句子
  → LLM添加标点
  → 分配回段落
  → 保存text_with_punct
  → API返回优先显示
```

## 配置项

### 环境变量
- `DEEPSEEK_API_KEY`: DeepSeek API密钥（必需）
- `DEEPSEEK_BASE_URL`: API基础URL（可选，默认官方API）
- `DEEPSEEK_MODEL`: 模型名称（可选，默认deepseek-chat）

### 可调参数
```python
# 在punctuation_restorer.py中
BATCH_SIZE = 100          # 每批处理数量
MERGE_THRESHOLD_MS = 3000 # 合并阈值（毫秒）
```

## 成本估算

### LLM调用成本
- **每批**: 100个段落
